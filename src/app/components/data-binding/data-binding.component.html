Dependency Injection

DI is a design pattern in Angular to manage how components and services acquire dependencies. It is a technique in which an object gets its dependencies from an external source rather than creating them itself.

Change Detection

Is angular's way of keeping data model and view in sync. Whenever angular think the view might need an update it runs change detection cycle.

The ChangeDetectionStrategy.OnPush setting is used to configure the change detection behavior for the component. With this strategy, change detection is triggered only when input properties change or when an event is emitted, improving the performance by reducing unnecessary checks and updates.

View Encapsulation

ViewEncapsulation is an Angular feature that allows you to control how styles are applied to a component's view. It determines whether styles defined in a component's CSS file are encapsulated to that component's view only or if they can affect other components as well. By default, Angular uses Emulated encapsulation, which means that component styles are scoped to the component's view and won't affect other components.
