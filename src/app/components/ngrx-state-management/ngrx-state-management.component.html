As Angular apps grow larger and more complex, managing the shared state (like user data, form inputs, filters, etc.) becomes difficult.

Ngrx helps by:

Making state predictable

Centralizing application state

Making it easier to debug and test

Store Centralized container for application state

Actions Plain objects that describe “what happened”

Reducers Pure functions that handle state transitions

Selectors Functions to read pieces of state from the store

Effects Side-effect handlers (e.g., API calls) that listen for actions and dispatch new ones

State Immutable data structure that holds app data

To create an action - createAction('[Counter] Increment');

To create a reducer -

export const counterReducer = createReducer( initialState, on(increment, state => state + 1), on(decrement, state => state - 1), on(reset, _ => 0) );

Import reducer in store moduele

Dispatching action

this.store.dispatch(increment());
